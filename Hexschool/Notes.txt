功能開發 → 建立 Server → 建立 option API 設置 cors 表頭資訊 → 404：無對應路由 → GET:所有 tode
→ POST：新增 todo → DELETE：刪除 todo → PATCH：編輯 todo → 部屬到 Heroku

1. 功能開發：針對 Web 應用程式的功能需求進行開發，可能需要使用到各種前端和後端的技術。
2. 建立 Server：建立一個 Web 伺服器，可以使用 Node.js、Java、Python 等語言和框架來實現。
3. 建立 option API 設置 cors 表頭資訊：對 Web 伺服器進行設置，以解決跨域請求的問題。
4. 404：無對應路由：在 Web 伺服器中設置路由，以實現對不同請求路徑的處理。
5. GET：所有 todo：實現一個 GET 請求路由，可以查詢所有的 todo 資料。
6. POST：新增 todo：實現一個 POST 請求路由，可以新增一條 todo 資料。
7. DELETE：刪除 todo：實現一個 DELETE 請求路由，可以刪除一條 todo 資料。
8. PATCH：編輯 todo：實現一個 PATCH 請求路由，可以編輯一條 todo 資料。
9. 部屬到 Heroku：將 Web 應用程式部屬到 Heroku 平台上，實現在雲端上運行。

----------------------------

Web應用程式開發流程可以概括為以下步驟：

1. 確定需求：明確定義需求，包括功能、性能、安全性等方面的需求，並為之制定計劃和時間表。
2. 設計架構：根據需求設計架構，包括系統架構、技術選型、資料庫設計等。
3. 編寫代碼：編寫前端和後端代碼，實現系統功能和頁面交互效果。
4. 建立測試：進行單元測試和集成測試，保證系統的正常運行和穩定性。
5. 代碼審查：進行代碼審查，確保代碼質量和標準。
6. 上線部署：將代碼部署到測試環境、預上線環境和正式環境，進行相應的測試和驗證。
7. 監控維護：在正式環境運行時，需要進行系統監控和維護，及時發現和處理問題，保證系統穩定運行。

總體來說，Web應用程式開發流程需要注重需求分析、設計架構、代碼編寫、測試和部署等多個環節，保證系統開發和上線的質量和效率。
